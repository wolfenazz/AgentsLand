## Item 6 ŌĆö Auto-save (30 min)

**Approach:** Add debounced auto-save via the existing `EditorView.updateListener`.

### Changes in `FileEditor.tsx`:

1. **Add state** ŌĆö `autoSave` boolean, default `true`. Add to `appStore` for persistence across sessions.
2. **Add debounce ref** ŌĆö `autoSaveTimerRef = useRef<ReturnType<typeof setTimeout> | null>(null)`
3. **Modify the `updateListener`** (line 281-284) ŌĆö When `update.docChanged` and `autoSave` is on, clear any existing timer and set a new 2s timeout that calls `handleSave()`.
4. **Cleanup** ŌĆö Clear the timer in the effect cleanup and on unmount.
5. **Add toolbar toggle** ŌĆö Small button in the toolbar area (next to Wrap) with an auto-save icon. Active state shown with blue highlight (same pattern as word wrap button).
6. **Persist setting** ŌĆö Add `autoSave: boolean` to `appStore.ts` persisted fields (alongside `theme`, `cliStatuses`, `selectedIdes`).

### Implementation detail:
```typescript
// Inside updateListener:
if (update.docChanged && currentFileRef.current) {
  callbacksRef.current.updateFileContent(currentFileRef.current, update.state.doc.toString());
  if (autoSave) {
    if (autoSaveTimerRef.current) clearTimeout(autoSaveTimerRef.current);
    autoSaveTimerRef.current = setTimeout(() => {
      callbacksRef.current.handleSave();
    }, 2000);
  }
}
```

## Item 14 ŌĆö Minimap (30 min)

**Current state:** `@replit/codemirror-minimap` v0.5.2 is installed in package.json but the import is commented out at line 18: `// minimap removed due to version conflict issues`.

**Approach:** Re-enable the minimap extension. The "version conflict" may have been resolved since the package was commented out. We'll add it as a compartment so it can be toggled.

### Changes in `FileEditor.tsx`:

1. **Import minimap extensions:**
   ```typescript
   import { minimap, showMinimap } from '@replit/codemirror-minimap';
   // or whatever the correct import is based on package API
   ```

2. **Add a `minimapCompartment`** ŌĆö Same pattern as `themeCompartment`/`wordWrapCompartment`:
   ```typescript
   const minimapCompartment = new Compartment();
   ```

3. **Add to extensions array** ŌĆö Include `minimapCompartment.of(showMinimap({ ... }))` or `minimapCompartment.of([])` based on default state.

4. **Add state** ŌĆö `showMinimap` boolean, default `true`. Add toggle in toolbar.

5. **Add reconfigure effect** ŌĆö Same pattern as word wrap:
   ```typescript
   useEffect(() => {
     if (!viewRef.current) return;
     viewRef.current.dispatch({
       effects: minimapCompartment.reconfigure(
         showMinimapSetting ? [minimap()] : []
       ),
     });
   }, [showMinimapSetting]);
   ```

6. **Add toolbar toggle** ŌĆö Button with minimap icon, same style as wrap toggle.

7. **Persist setting** ŌĆö Add `showMinimap: boolean` to `appStore` persisted fields.

**Fallback:** If `@replit/codemirror-minimap` v0.5.2 still has version conflicts with the current CodeMirror packages, we'll either:
- Update to a newer version of `@replit/codemirror-minimap` that's compatible
- Or use a simple gutter-based minimap approach instead
