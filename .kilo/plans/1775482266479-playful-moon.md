# Plan: Discord Rich Presence Integration

## Overview
Add Discord Rich Presence to YzPzCode using the `discord-rich-presence` Rust crate on the backend, with dynamic status updates driven by the frontend. Toggle will be placed under **Settings > Appearance**.

## Client ID
`1490701639362023464` (from Discord Dev Portal, per `docs/discord.md`)

---

## Step 1: Add Rust Dependency

**File:** `app/src-tauri/Cargo.toml`

Add `discord-rich-presence = "0.2"` to `[dependencies]`.

---

## Step 2: Create Discord RPC Rust Module

**New File:** `app/src-tauri/src/discord.rs`

Create a `DiscordRpcManager` struct that:
- Stores a `discord_rich_presence::DiscordIpcClient` behind `Arc<Mutex<>>`
- Provides `connect()` — connects to Discord IPC, sets initial activity
- Provides `set_activity(details, state)` — updates the Rich Presence
- Provides `clear_activity()` — clears the current presence
- Provides `reconnect()` — attempts reconnection if disconnected
- Implements `Clone` (via Arc)
- All methods are fallible but errors are logged, never propagated to crash

The default activity when connected:
```
details: "Using YzPzCode"
state: "Idle"
large_image_key: "apppic"
large_image_text: "YzPzCode"
start_timestamp: <connect time>
```

## Step 3: Create Discord Tauri Commands

**New File:** `app/src-tauri/src/commands/discord_commands.rs`

Tauri commands (thin wrappers following project pattern):
- `update_discord_presence(details: String, state: String, large_image_key: Option<String>)` — updates activity
- `clear_discord_presence()` — clears activity
- `set_discord_presence_enabled(enabled: bool)` — connects or disconnects RPC

**File:** `app/src-tauri/src/commands/mod.rs` — add `mod discord_commands;` and `pub use discord_commands::*;`

## Step 6: Create useDiscordPresence Hook

**New File:** `app/src/hooks/useDiscordPresence.ts`

A React hook that:
- Reads `discordRichPresence` from the store
- On mount or toggle change, calls `invoke('set_discord_presence_enabled', { enabled })`
- Watches `view`, `currentWorkspace`, `activeFilePath`, `activeView`, `openWorkspaces` to build dynamic status
- Calls `invoke('update_discord_presence', ...)` whenever relevant state changes
- Dynamic presence logic:
  - **Setup view**: details = "Using YzPzCode", state = "Setting up workspace"
  - **Workspace view, terminal active**: details = "In {workspace.name}", state = "Running terminals"
  - **Workspace view, editor active**: details = "In {workspace.name}", state = "Editing {filename}"
  - **Settings view**: details = "Using YzPzCode", state = "Adjusting settings"
  - **Docs view**: details = "Using YzPzCode", state = "Reading docs"
- Debounces updates (500ms) to avoid flooding Discord IPC

---

## Step 7: Integrate Hook in App.tsx

**File:** `app/src/App.tsx`

Import and call `useDiscordPresence()` in the `App` component. This ensures the hook is active for the entire app lifecycle.

---

## Step 8: Add Toggle to Appearance Settings

**File:** `app/src/components/settings/sections/SettingsAppearance.tsx`

Add a toggle row for "Discord Rich Presence" with the `discordRichPresence` state from the store. The toggle should:
- Show an info label: "Show YzPzCode activity status on Discord"
- Toggle the `discordRichPresence` store value
- The hook in App.tsx handles the actual connect/disconnect via the store change
