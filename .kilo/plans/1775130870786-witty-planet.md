# Plan: Fix Terminal Resize Black Space

## Problem
After adding dnd-kit drag-and-drop, terminal panes no longer fill their grid cells when resized. Dragging a divider changes the CSS Grid row/column sizes, but the terminals inside don't grow/shrink to match, leaving black space.

## Root Cause
`SortableTerminalPane.tsx` line 35 — the wrapper `<div>` added by dnd-kit has no `h-full` class:

```tsx
<div ref={setNodeRef} style={style} {...attributes}>  // ← no h-full
  <TerminalPane ... />  // ← has h-full, but parent has no height
</div>
```

The height chain is broken:
1. Grid cell → gets height from CSS Grid row ✓
2. SortableTerminalPane wrapper → **no `h-full`** → doesn't stretch ✗
3. TerminalPane → `h-full` resolves to 0 because parent has no height ✗

## Fix

### File: `app/src/components/workspace/SortableTerminalPane.tsx`
- Add `className="h-full"` to the wrapper div at line 35
- This makes the wrapper stretch to fill its grid cell, and TerminalPane's `h-full` then resolves correctly

### File: `app/src/components/workspace/TerminalGrid.tsx` — Enhanced Resize
- Grid cell containers already stretch via CSS Grid, no changes needed there
- Verify the gap calculation in divider positioning accounts for the 6px gap correctly

## Single Change
```diff
// SortableTerminalPane.tsx line 35
-    <div ref={setNodeRef} style={style} {...attributes}>
+    <div ref={setNodeRef} style={style} className="h-full" {...attributes}>
```
